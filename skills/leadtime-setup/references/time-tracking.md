# Chapter F: Time Tracking

Use this chapter when the customer wants attendance, working time, time booking, billable targets, or time reports.

## Explain The Model

Leadtime separates:

- **Attendance**: when a person is working or present.
- **Time bookings**: what the person worked on, usually linked to tasks/projects and activities.
- **Billable targets**: expected chargeable hours per person/day.
- **Working time models**: weekly hours, schedules, part-time settings, and valid-from dates.

## Configuration Areas

Read the current schema/settings first, then configure:

- Attendance mode, when available. Some workspaces use active clock-in/out; others calculate attendance from contractual working time.
- Time booking targets and allowed projects/tasks.
- Activities used during time booking.
- Employee working time entries.
- Vacation and absence settings.
- Chargeable daily goal per employee when relevant.

Chargeable daily targets are usually employee-specific, not a workspace default. Load employee IDs through MCP, but show employee names to the user.

## Review Checkpoint

After configuration, provide deeplinks to the team calendar, staff settings, or time tracking pages as appropriate. Ask the user to review one full-time and one part-time employee if applicable.

When core setup is complete, recommend a short employee briefing, the knowledge base, and ongoing `/leadtime-coach` usage for future questions.
